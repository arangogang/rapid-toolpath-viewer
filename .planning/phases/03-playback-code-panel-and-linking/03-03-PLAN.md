**Part B: Triad geometry builder (geometry_builder.py)**

Add `build_triad_vertices()` function and a `triad_verts` field to GeometryBuffers:

1. Add `triad_verts: np.ndarray` field to `GeometryBuffers` dataclass (shape (N, 6) float32, same interleaved layout)
2. Add function:
   ```python
   def build_triad_vertices(
       moves: list[MoveInstruction], length: float = 15.0
   ) -> np.ndarray:
       """Build TCP orientation triad vertices for all Cartesian moves.

       For each move with has_cartesian=True and a valid orient quaternion,
       generates 6 vertices (3 axis lines: R=X, G=Y, B=Z).

       ABB quaternion convention: [q1,q2,q3,q4] = [w,x,y,z]
       pyrr expects: [x,y,z,w]
       """
   ```
   - Iterate moves, skip non-Cartesian
   - For each move: get pos from move.target.pos, orient from move.target.orient
   - Convert ABB [w,x,y,z] to pyrr [x,y,z,w]: `pyrr_quat = np.array([orient[1], orient[2], orient[3], orient[0]])`
   - Use `pyrr.matrix33.create_from_quaternion(pyrr_quat)` to get 3x3 rotation matrix
   - For each of 3 axes (X=red, Y=green, Z=blue): axis_end = pos + rot @ (unit_vec * length)
   - Emit 2 vertices per axis: [pos, color] and [axis_end, color]
   - Return np.array shape (N*6, 6) float32, or (0, 6) if empty
3. Update `build_geometry()` to also call `build_triad_vertices(result.moves)` and include result in GeometryBuffers
