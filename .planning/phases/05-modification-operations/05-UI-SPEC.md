## Component Inventory

### Modified Widgets (Phase 5 changes to existing)

#### PropertyPanel (convert from read-only to editable)

**Header section** (unchanged):
- `QLabel` bold 9pt: "{N} point(s) selected" or "No selection"

**Position group** (read-only display, unchanged):
- `QFormLayout` with QLabel rows: X, Y, Z showing current position at 3 decimal places
- Format: `{value:.3f}` (e.g., "1234.567")

**Offset group** (NEW -- inserted below Position group):
- `QGroupBox("Offset")` with `QFormLayout`
- Three `QLineEdit` inputs: dX, dY, dZ
  - Placeholder text: "0.0" in each field
  - Validator: `QDoubleValidator` allowing negative values, 3 decimal places
  - Width: fill available form width (FieldGrowthPolicy.AllNonFixedFieldsGrow)
  - Tab order: dX -> dY -> dZ -> Apply Offset button
- `QPushButton("Apply Offset")`
  - Enabled when: at least one point selected AND at least one offset field is non-empty/non-zero
  - Disabled styling: Qt default grayed-out
  - Behavior: commits offset to all selected points as one QUndoCommand (D-12)
  - After apply: fields retain values (D-05), selection unchanged

**Motion group** (convert labels to inputs):
- `QFormLayout` with:
  - `QLabel` row: Type (read-only, always label, shows MoveL/MoveJ/MoveC/MoveAbsJ)
  - `QLineEdit` row: Speed (free-text, e.g., "v500", "vmax") (D-02)
  - `QLineEdit` row: Zone (free-text, e.g., "z10", "fine") (D-02)
- Commit: Enter key press creates QUndoCommand immediately (D-04)
- Multi-select: shows first selected point's values; edit overwrites ALL selected (D-13)

**Laser group** (convert label to combo):
- `QFormLayout` with:
  - `QComboBox` row: State with items ["ON", "OFF"] (D-03)
- Commit: `currentIndexChanged` creates QUndoCommand immediately (D-04)
- Multi-select: shows first selected point's value; change overwrites ALL selected (D-13)

**Action buttons section** (NEW -- bottom of panel):
- `QPushButton("Insert After")`
  - Enabled when: exactly 1 point selected (D-09, D-15)
  - Behavior: creates new point at selected.pos + offset, new point becomes selected (D-10)
  - Uses offset values from Offset group (D-09)
- `QPushButton("Delete")`
  - Enabled when: at least 1 point selected (D-06)
  - Keyboard shortcut: Del key (D-06)
  - Styling: `setStyleSheet("QPushButton { background-color: #CC3333; color: white; padding: 4px 8px; }")` (D-06)
  - Behavior: opens delete confirmation dialog (D-07)

#### Layout order (top to bottom within PropertyPanel scroll area):
1. Header label
2. Position group (read-only QLabels)
3. Offset group (dX/dY/dZ inputs + Apply Offset button)
4. Motion group (Type label + Speed/Zone inputs)
5. Laser group (State combo)
6. Stretch spacer
7. Insert After button
8. Delete button

### New Dialogs

#### Delete Confirmation Dialog (D-07)

- Type: `QMessageBox` with custom buttons
- Title: "Delete Point(s)"
- Text: "Delete {N} point(s)?"
- Icon: `QMessageBox.Icon.Warning`
- Buttons (3 custom):
  - `QPushButton("Reconnect")` -- path stays continuous (D-08)
  - `QPushButton("Break")` -- laser-off gap inserted (D-08)
  - `QPushButton("Cancel")` -- standard cancel role
- Default button: Cancel (safe default)
- Escape key: Cancel

### Existing Widgets (no Phase 5 changes)

- `ToolpathGLWidget`: receives `update_scene()` call after edits (existing pattern)
- `CodePanel`: no changes -- code panel remains read-only
- `PlaybackToolbar`: no changes
- `MainWindow`: adds signal wiring for PropertyPanel edit signals to EditModel

## Interaction Contracts

### Coordinate Offset (MOD-01)

| Step | Actor | Action | System Response |
|------|-------|--------|-----------------|
| 1 | User | Selects 1+ waypoints (click/Shift/Ctrl in 3D) | PropertyPanel shows point properties, offset fields enabled |
| 2 | User | Types values in dX/dY/dZ fields | Apply Offset button becomes enabled |
| 3 | User | Clicks "Apply Offset" or presses Enter in offset field | Offset applied to all selected points. 3D view updates immediately. Title bar shows dirty asterisk. Offset fields retain values (D-05). |
| 4 | User | Clicks "Apply Offset" again | Same offset applied again (cumulative). Enables rapid repeated offsets (D-05). |
| 5 | User | Presses Ctrl+Z | Last offset undone. 3D view reverts. Fields retain current values. |

### Property Edit (MOD-02)

| Step | Actor | Action | System Response |
|------|-------|--------|-----------------|
| 1 | User | Selects 1+ waypoints | PropertyPanel shows Speed/Zone inputs, Laser combo with current values |
| 2 | User | Changes Speed field text and presses Enter | New speed value applied to all selected points. QUndoCommand created. |
| 3 | User | Changes Laser combo from ON to OFF | Laser state applied to all selected points immediately (D-04). QUndoCommand created. |

### Delete (MOD-03)

| Step | Actor | Action | System Response |
|------|-------|--------|-----------------|
| 1 | User | Selects 1+ waypoints | Delete button enabled |
| 2 | User | Clicks Delete button or presses Del key | Confirmation dialog appears: "Delete N point(s)?" |
| 3a | User | Clicks "Reconnect" | Points deleted. Path reconnects through remaining points. 3D view updates. Selection cleared. |
| 3b | User | Clicks "Break" | Points deleted. Laser-off gap inserted. 3D view updates. Selection cleared. |
| 3c | User | Clicks "Cancel" or presses Escape | Dialog dismissed. No changes. |
| 4 | User | Presses Ctrl+Z | Deletion undone. Points restored. 3D view reverts. |

### Continuous Insertion (MOD-04)

| Step | Actor | Action | System Response |
|------|-------|--------|-----------------|
| 1 | User | Selects exactly 1 waypoint | Insert After button enabled |
| 2 | User | Types offset values in dX/dY/dZ | Values ready for insertion |
| 3 | User | Clicks "Insert After" | New point created at selected.pos + offset. Properties copied from source (D-10). New point becomes selected (D-10). |
| 4 | User | Clicks "Insert After" again | Another new point created at new_point.pos + offset. Chained insertion (D-11). |
| 5 | User | Clicks elsewhere or selects different point | Insertion chain ends naturally (D-11). |

## State Visibility

| State | Visual Indicator |
|-------|-----------------|
| No selection | PropertyPanel header: "No selection". All fields show "--". Offset/Delete/Insert disabled. |
| Single selection | Header: "1 point selected". All fields populated. All actions enabled. |
| Multi-selection | Header: "{N} points selected". Fields show first selected point's values. Insert After disabled (D-15). |
| Dirty (unsaved edits) | Title bar prefix: "* " before filename (existing from Phase 4). |
| Clean (no edits or after save) | No asterisk in title bar. |
| Delete button hover | Qt default hover effect on red button. |
