3. Create `src/rapid_viewer/export/mod_writer.py` with module docstring. Pure Python, no Qt dependency. The module implements:

**Public function signature:**
```python
def export_mod(
    source_text: str,
    points: list[EditPoint],
    targets: dict[str, RobTarget],
) -> str:
```

**Algorithm (line-level source text patching):**
a. Split `source_text` into lines (preserving line endings).
b. Build a patch list by iterating all points:
   - For each non-deleted, non-inserted point: compare EditPoint fields against `point.original` to detect changes.
     - Position changed + named target (name not starting with `<` and not starting with `Offs(`): REPLACE the robtarget declaration line. Use `targets[target_name].source_line` to find the declaration. For multiline declarations, scan from that line forward until a line containing `;` to find the full extent. Regex-replace the first `[[x,y,z]` bracket group with new values formatted as `f"{val:.6g}"`.
     - Position changed + Offs() target (name starts with `Offs(`): Compute delta = point.pos - point.original.target.pos. Add delta to the original Offs() numeric args. Parse the original Offs() call from the move line using regex, identify which args are numeric vs variable, add delta to numeric args only. Replace in the move line.
     - Position changed + inline target (name == `<inline>`): Regex-replace the first `[[x,y,z]` group in the move instruction line.
     - Speed changed: Regex-replace the speed token in the move line. Pattern: after the target argument and first comma, the next token is speed.
     - Zone changed: Regex-replace the zone token (after speed and comma).
   - For deleted points (`point.deleted == True`): DELETE the source line (prefix with `! [DELETED] `).
   - For inserted points (`point.is_inserted == True`): INSERT_AFTER the source line. Generate a MoveL line with inline robtarget: `    MoveL [[x,y,z],[q1,q2,q3,q4],[cf1,cf4,cf6,cfx],[eax_a,...,eax_f]],speed,zone,tool \WObj:=wobj;` Always generate as MoveL regardless of original move type (MoveC makes no sense for arbitrary inserts).

c. Sort patches by line number DESCENDING (highest first) so index shifts don't affect lower patches.

d. Apply patches to the lines array.

e. Join and return.

**Helper functions to create:**
- `_patch_robtarget_pos(lines: list[str], start_line_idx: int, new_pos: np.ndarray) -> None` -- patches a (possibly multiline) robtarget declaration's position bracket group. Finds the extent by scanning for `;`. Joins the extent into one string, regex-replaces `[[x,y,z]` with new values, then splits back and replaces the lines.
- `_patch_offs_args(line: str, delta: np.ndarray) -> str` -- parses the Offs() call, adds delta to numeric args (preserving variable args like `zLayer`), returns patched line.
- `_patch_inline_pos(line: str, new_pos: np.ndarray) -> str` -- regex-replaces the first `[[x,y,z]` in a move line containing inline target data.
- `_patch_speed_zone(line: str, new_speed: str | None, new_zone: str | None) -> str` -- replaces speed and/or zone tokens in a move instruction line. Uses regex: `(Move\w+\s+)(.+?)(,\s*)(\w+)(\s*,\s*)(\w+)(\s*,.*)` where group 4 is speed, group 6 is zone. Handle the target arg (group 2) carefully -- it may contain commas inside brackets.
- `_generate_move_line(point: EditPoint, indent: str) -> str` -- generates a RAPID MoveL line with inline robtarget for inserted points. Format numbers with `f"{val:.6g}"` for clean output.

**Format number values** using `f"{val:.6g}"` which gives clean output (no trailing zeros, scientific notation only for very large/small).

**Laser on/off changes are NOT exported as SetDO modifications** -- the original SetDO lines are preserved as-is. This is a documented limitation per RESEARCH.md decision 4.

**Track patched robtarget declarations** to handle the shared-target case: if a named target was already patched (by a previous point), skip subsequent patches to the same declaration line. Use a `patched_lines: set[int]` to track.
