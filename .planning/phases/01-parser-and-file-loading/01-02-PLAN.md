  <implementation>
    Follow the two-pass architecture from RESEARCH.md exactly:

    1. `tokenize_statements(source: str) -> list[tuple[str, int]]`:
       - Iterate lines with enumerate(source.splitlines(), start=1)
       - Strip RAPID comments: split on '!' and take first part
       - Split on ';' to detect statement boundaries
       - Accumulate fragments until ';' found, tracking start_line as the line where first non-empty content appeared
       - Return list of (normalized_statement, start_line) tuples

    2. `parse_robtarget_data(bracket_str: str) -> tuple[np.ndarray, np.ndarray, tuple, tuple]`:
       - Use RE_BRACKET_GROUP.findall(bracket_str) to extract inner bracket contents
       - Expect 4 groups: pos(3 floats), orient(4 floats), confdata(4 ints), extjoint(6 floats)
       - Parse numbers supporting scientific notation (9E+09)
       - Return (pos_array, orient_array, confdata_tuple, extjoint_tuple)

    3. `try_parse_robtarget_decl(stmt: str, line_num: int) -> RobTarget | None`:
       - Match RE_ROBTARGET_DECL against stmt
       - If match, extract name (group 1) and bracket data (group 2)
       - Call parse_robtarget_data, construct RobTarget

    4. `try_parse_jointtarget_decl(stmt: str, line_num: int) -> JointTarget | None`:
       - Match RE_JOINTTARGET_DECL, extract name and bracket data
       - Parse 2 bracket groups: robax (6 floats) and extax (6 floats)

    5. `resolve_target_ref(ref_str: str, targets: dict, line_num: int) -> RobTarget | None`:
       - If ref_str matches RE_OFFS: extract base_name, dx, dy, dz. Look up base in targets. Return new RobTarget with pos = base.pos + [dx, dy, dz], same orient/confdata/extjoint, name=f"Offs({base_name},{dx},{dy},{dz})"
       - If ref_str starts with '[': parse as inline robtarget
       - Else: look up ref_str in targets dict, return the RobTarget or None

    6. `try_parse_move(stmt, line_num, targets, joint_targets) -> MoveInstruction | None`:
       - Try RE_MOVEL, RE_MOVEJ, RE_MOVEC, RE_MOVEABSJ in order
       - Extract groups per pattern (see interfaces above)
       - For MoveC: resolve BOTH circle_point and endpoint
       - For MoveAbsJ: look up in joint_targets, set has_cartesian=False, target=None
       - Extract optional \WObj via RE_WOBJ (default "wobj0" if not present)
       - Return MoveInstruction

    7. `parse_module(source: str) -> ParseResult`:
       - Call tokenize_statements(source)
       - Pass 1: iterate statements, call try_parse_robtarget_decl and try_parse_jointtarget_decl, build lookup dicts
       - Also extract module name via RE_MODULE and proc names via RE_PROC into lists
       - Pass 2: iterate statements, call try_parse_move with lookup dicts
       - Return ParseResult with all collected data

    8. `read_mod_file(path: Path) -> str`:
       - Try UTF-8, catch UnicodeDecodeError, fall back to latin-1

    REFACTOR phase:
    - Ensure all functions have docstrings
    - Extract any duplicated parsing logic
  </implementation>
